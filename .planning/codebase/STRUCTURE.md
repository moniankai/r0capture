## Directory Purposes

**frida_hooks/:**
- Purpose: 存放所有 Frida JavaScript Hook 脚本
- Contains: Java 层 Hook（TTVideoEngine、OkHttp、ExoPlayer）、Native 层 Hook（AES、DRM）、反检测脚本
- Key files: 
  - `ttengine_all.js` — Hook TTVideoEngine 所有关键方法
  - `aes_hook.js` — Hook javax.crypto.Cipher 捕获 AES 密钥和 IV
  - `okhttp_hook.js` — Hook OkHttp 记录 HTTP 请求
  - `anti_detection.js` — 绕过 Frida 检测
  - `exoplayer_hook.js` — Hook ExoPlayer 播放器
  - `mediacodec_java.js` — Hook MediaCodec 解码器

**scripts/:**
- Purpose: 核心 Python 工具脚本集合
- Contains: 下载器、解密器、Hook 管理、UI 自动化、审计工具
- Key files:
  - `download_drama.py` — 主工作流：Frida Hook + 下载 + 解密（92KB，最复杂模块）
  - `drama_download_common.py` — 共享工具：UI 解析、会话校验、文件名生成
  - `decrypt_video.py` — MP4 CENC 解密器（AES-CTR-128）
  - `batch_manager.py` — 多线程批量下载队列
  - `hls_downloader.py` — HLS/M3U8 下载器（支持 AES-128 解密）
  - `pcap_parser.py` — PCAP 文件解析器
  - `pull_cache.py` — 从 App 缓存拉取视频
  - `audit_drama_downloads.py` — 离线审计：缺集检测、重复识别
  - `preprocess_video.py` — LLM 预处理：关键帧 + Whisper ASR
  - `check_environment.py` — 环境检测和 Frida Server 安装
  - `capture_key.py` — 专用 AES 密钥捕获工具
  - `capture_and_download.py` — 通用捕获与下载流程

**tests/:**
- Purpose: 单元测试和集成测试
- Contains: pytest 测试用例
- Key files:
  - `test_download_drama.py` — 测试 UI 解析、会话校验、文件名生成
  - `test_audit_drama_downloads.py` — 测试审计功能

**docs/:**
- Purpose: 技术文档和设计规划
- Contains: 技术指南、逆向分析记录、实现计划
- Key files:
  - `honguo_decrypt_guide.md` — 红果短剧解密完整技术方案
  - `plans/` — 设计文档和实现计划（markdown）
  - `forensics/` — 逆向分析记录

**videos/:**
- Purpose: 下载视频输出目录（运行时生成）
- Contains: 按剧名组织的子目录，每个子目录包含解密后的 MP4、元数据 JSON、会话日志
- Structure:
  ```
  videos/<剧名>/
  ├── episode_001_<8位vid>.mp4
  ├── meta_ep001_<8位vid>.json
  ├── session_manifest.jsonl
  └── llm_ready/  # --preprocess 时生成
  ```

**tasks/:**
- Purpose: 任务跟踪和经验教训
- Contains: TODO 列表、经验教训记录
- Key files:
  - `todo.md` — 当前任务和阶段规划
  - `lessons.md` — 经验教训和自我改进记录

**.planning/:**
- Purpose: GSD 规划文档（codebase 映射）
- Contains: 架构分析、技术栈、约定、测试模式
- Generated by: `/gsd-map-codebase` 命令

**openspec/:**
- Purpose: OpenSpec 规范文档
- Contains: 变更规范、批量管理器规范、Frida 环境规范

## Naming Conventions

**Files:**
- Python 脚本：`snake_case.py`（例如 `download_drama.py`、`decrypt_video.py`）
- JavaScript Hook：`snake_case.js`（例如 `aes_hook.js`、`ttengine_all.js`）
- 测试文件：`test_<module>.py`（例如 `test_download_drama.py`）
- 文档：`kebab-case.md` 或 `UPPERCASE.md`（例如 `honguo_decrypt_guide.md`、`README.md`）

**Directories:**
- 全小写 + 下划线：`frida_hooks/`、`app_data/`
- 隐藏目录：`.planning/`、`.tmp/`、`.worktrees/`

**Variables:**
- Python：`snake_case`（例如 `video_id`、`aes_keys`、`capture_state`）
- JavaScript：`camelCase`（例如 `videoModel`、`aesKey`、`dumpObj`）
- 常量：`UPPER_SNAKE_CASE`（例如 `APP_PACKAGE`、`HOOK_TIMEOUT`、`FRIDA_VERSION_MAP`）

**Functions:**
- Python：`snake_case`（例如 `parse_ui_context()`、`decrypt_mp4()`、`run_adb()`）
- JavaScript：`camelCase`（例如 `dumpObj()`、`bytesToHex()`、`findFieldInHierarchy()`）

**Classes:**
- Python：`PascalCase`（例如 `CaptureState`、`SessionValidationState`、`UIContext`、`BatchManager`）

## Where to Add New Code

**New Feature:**
- Primary code: `scripts/<feature_name>.py`
- Tests: `tests/test_<feature_name>.py`
- Documentation: `docs/<feature_name>.md` 或更新 `README.md`

**New Hook:**
- Implementation: `frida_hooks/<target>_hook.js`
- Integration: 在 `scripts/download_drama.py` 的 `COMBINED_HOOK` 中引用或作为独立 Hook 加载
- Documentation: 在 `docs/forensics/` 中记录逆向分析过程

**New Downloader:**
- Implementation: `scripts/<protocol>_downloader.py`（参考 `hls_downloader.py`）
- Integration: 在 `batch_manager.py` 中添加格式检测和调度逻辑

**New Utility:**
- Shared helpers: `scripts/drama_download_common.py`（如果与短剧下载相关）
- Generic helpers: 创建新的 `scripts/<utility_name>.py`

**New Test:**
- Unit tests: `tests/test_<module>.py`
- Test fixtures: 在测试文件内定义 XML/JSON 样本（参考 `test_download_drama.py` 中的 `SAMPLE_UI_XML`）

**New Documentation:**
- Technical guides: `docs/<topic>.md`
- Implementation plans: `docs/plans/<date>-<topic>.md`
- Forensics notes: `docs/forensics/<topic>.md`

## File Naming Patterns

**Downloaded Videos:**
- Pattern: `<剧名>_episode_<集数>_<8位vid>.mp4`
- Example: `十八岁太奶奶驾到，重整家族荣耀_episode_001_a1b2c3d4.mp4`
- Location: `videos/<剧名>/`

**Metadata Files:**
- Pattern: `meta_ep<集数>_<8位vid>.json`
- Example: `meta_ep001_a1b2c3d4.json`
- Location: `videos/<剧名>/`

**Session Logs:**
- Pattern: `session_manifest.jsonl`
- Format: JSON Lines（每行一个 JSON 对象）
- Location: `videos/<剧名>/`

**PCAP Files:**
- Pattern: `<package>_<timestamp>.pcap` 或用户指定名称
- Example: `com_phoenix_read_1775792839.pcap`
- Location: 项目根目录

**Temporary Files:**
- Pattern: `<prefix>_<timestamp>.log` 或 `<prefix>_<timestamp>.json`
- Example: `honguo_capture_1775794495.log`
- Location: 项目根目录或 `.tmp/`

## Import Patterns

**Internal Imports:**
- 脚本间导入：`from scripts.<module> import <name>`
- 示例：`from scripts.decrypt_video import decrypt_mp4, fix_metadata`
- 路径调整：`sys.path.insert(0, str(Path(__file__).parent.parent))`（在 scripts/ 内的脚本中）

**External Dependencies:**
- Frida：`import frida`
- HTTP 请求：`import requests`
- 日志：`from loguru import logger`
- 加密：`from Crypto.Cipher import AES`
- 进度条：`from tqdm import tqdm`
- HLS 解析：`import m3u8`

**Standard Library:**
- 文件操作：`from pathlib import Path`
- 子进程：`import subprocess`
- 数据类：`from dataclasses import dataclass, field`
- 类型提示：`from typing import Optional, Any`
- XML 解析：`import xml.etree.ElementTree as ET`

## Code Organization Principles

**Separation of Concerns:**
- Hook 脚本（JavaScript）与 Python 协调层分离
- UI 解析逻辑（`drama_download_common.py`）与主工作流（`download_drama.py`）分离
- 解密算法（`decrypt_video.py`）可独立运行

**Reusability:**
- `drama_download_common.py` 提供共享工具函数
- `batch_manager.py` 可用于任意视频 URL 批量下载
- `hls_downloader.py` 支持通用 HLS 流下载

**Testability:**
- UI 解析函数接受 XML 字符串参数，无需真实设备
- 会话校验逻辑纯函数化，易于单元测试
- 测试文件包含完整的 XML 样本数据

**Modularity:**
- 每个脚本可独立运行（`decrypt_video.py`、`audit_drama_downloads.py`、`pull_cache.py`）
- 主工作流通过导入组合各模块功能
